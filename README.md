### Structure

`<type>: <description>`

### Type

`feat` A new feature
<br>
`fix` A bug fix
<br>
`docs` Documentation only changes
<br>
`style` Changes that do not affect the meaning of the code (whitespace, formatting, missing semicolons, ect)
<br>
`refactor` A code change that neither fixes a bug nor adds a feature
<br>
`perf` A code change that improves performance
<br>
`test` Adding missing tests or correcting existing tests
<br>
`build` Changes that affect the build system or dependencies (example scopes: bun, gulp, npm)
<br>
`ci` Changes to our CI configuration files and scripts (example: Travis, Circle, BrowserStack, SauceLabs)
<br>
`chore` Other changes that don't modify src or test files
<br>
`revert` Revert a previous commit
